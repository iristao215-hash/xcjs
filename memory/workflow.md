# AI 工作流程手册

> 这是项目的"操作系统"。任何 AI（Claude / ChatGPT / Gemini…）开始为这个长篇互动叙事写作前，**必须先读完本文件**。
>
> 项目语境：中文长篇互动叙事《新网球王子》同人，主角陶雨（中国留学生）× 男主幸村精市。穿越者设定：陶雨与斋藤侑子双穿越者，斋藤是女主竞争者/反派。

## 🚨 写作铁律（最高优先级 · 违反即作废重写）

### 铁律一：**严禁凭记忆引用任何具体过往细节**

凡涉及"过去某天某人说了什么 / 做了什么 / 是否在场 / 时间精确点 / 动作细节"——
**必须**：
1. 先查 `memory/events-index.md` 或 `memory/timeline-index.md` 定位日期 → 拿到文件名
2. **用 Read 工具完整读取该文件**（含 frontmatter）
3. 在回应里**显式声明**："已读 `volumes/volN/MM-DD_xxx.md`"
4. 严格按文件内容引用细节

**若未读对应文件，必须写**："我没有读过那段·所以不写具体细节" 或 "请允许我先查文件再写"。

绝对禁止：根据 outline/header summary/grep 片段脑补细节后写出"具体的旧场景动作和台词"。

### 铁律二：**严禁仅靠 grep 拼凑历史信息**

grep 只能用来**定位**（找到该读哪个文件），不能用来**摘录**。读全文 → 引用 → 在显式声明里说明读过哪个文件。

⚠️ **grep 前必查 `memory/_专名表.md`**——把搜索关键词展开同义词后再 grep·否则会漏（如搜"陶雨"漏掉"阿雨"·搜"切原"漏掉"赤也"）。

### 铁律三：**frontmatter 是反事实警示的最高来源**

每个 day 文件顶部的 frontmatter 都有 `在场` / `不在场（重要⚠️）` 字段。这是防止 AI 把不在场的人写进场的最关键护栏。**任何"某人当时在不在"的判断必须以 frontmatter 为准**。

### 铁律四：**写完场景后·必须做"矛盾自检"**

每个场景写完·**提交给用户前**·AI 必须按以下 checklist 逐条自检·**任何一项不通过 → 当场标注 ⚠️ 让用户审**：

#### ✅ 矛盾自检 checklist（**每场必走·一项不漏**）

**A. 在场/不在场核对**
- [ ] 我写的"在场人物"·是否每一个都有 day 文件 frontmatter 支持？
- [ ] 我引用的过去某天某人在场·是否查过那个 day 文件的 `在场` 字段确认？
- [ ] 我引用的某人"那时也在"·是否查过 `不在场（重要⚠️）`字段确认 ta 没被排除？
  - ⚠️ 典型错例：12.18 下午奶奶**不在家**（茶道教室）·写她在玄关迎客 = 矛盾
  - ⚠️ 典型错例：2.2 决赛日**陶雨不在墨尔本**（在神奈川远程关注）·写她现场鼓励 = 矛盾

**B. 时间线连续性**
- [ ] 我写的场景时间·是否跟最近一个 day 文件接续合理？（如距上一场超过 1 周·必说明这周发生了什么）
- [ ] 我引用的"上次某事"·距当前场景时间是否合理？
- [ ] 角色当前年龄/年级是否匹配？（如龙马 12.24 满 15·之后场景他是 15 岁·之前是 14 岁）

**C. 好感度数值核对**
- [ ] 场景里如有好感度变化·**起始值是否跟 `memory/affinity.md` 当前最新值一致**？
- [ ] 数值计算是否按"原始权重 × 系数 = 实际变化"格式？
- [ ] 涉及多角色变化·每个角色都单独列出？

**D. 关键金句精确性**
- [ ] 我让某人说的"过去原话"·是否跟 day 文件的 `关键金句` 字段一字一致？
- [ ] 没有把不同场景的台词混搭？（典型错例：把 12.18 奶奶的话拼到 12.7 别人嘴里）

**E. 物品/设定精确性**
- [ ] 我提到的物品·是否符合 `memory/settings/重要物品.md` 描述？
- [ ] 地点/座位/住址·是否符合 `memory/settings/地理与住所.md`？
- [ ] 称呼是否正确（切原叫陶雨"雨姐"·美咲叫"送项链的姐姐"·南次郎叫"小雨"等）？

**F. 暗线状态核对**
- [ ] 涉及佐伯·当前是否还在"已读不回千叶崎"状态？她的转折决定我没替她做吗？
- [ ] 涉及仁王×锥生·当前是否"互相明白心意"状态？没写成还在追？
- [ ] 涉及柳·暗恋千叶崎暗线没主动写出来吗？
- [ ] 涉及陶雨家·乐乐弟弟·**幸村当前不知道**·没让幸村提及吗？
- [ ] 涉及斋藤·崩溃值 28/100·状态没倒退？

**G. 写作禁忌核对**
- [ ] 陶雨不只是"直接"·有温柔有韧的双重性？
- [ ] 幸村不是事事顺从·有棱角？
- [ ] 幸村可以输·没写成"必赢男主"？
- [ ] 斋藤不浅薄·是高度自洽的天命女主叙事？
- [ ] 迹部不傲娇·觉得好笑正面笑·即使输了昂着头走？
- [ ] 真田已退让为深度保护·没写成对陶雨暧昧？

**H. 触发规则覆盖**
- [ ] 学校场景·有没有意识到斋藤/千叶崎在背景？
- [ ] 幸村家场景·有没有妈妈对千叶崎旧倾向的暗线？
- [ ] 涉及不二·幸村特殊醋点考虑了吗？
- [ ] 涉及手冢/龙马/果果·陶雨可在他们面前崩溃的安全感写出来了吗？

#### 自检发现矛盾时·处理方式

1. **明确报警**：在场景末尾用 ⚠️ 标出疑点·让用户决定改还是保留
2. **优先采信文件**：如果记忆和文件冲突·**永远以文件为准**
3. **未读文件就编**：直接标"我没有读过 vol2/12-18_告白.md·这段无法确认·暂留空待用户补"·**绝不脑补**

#### 自检最后一题
- [ ] **我在回应里显式列出了本场读过的所有 day 文件名·让用户能审计吗？**

### 文件命名约定

- **单日文件**：`MM-DD_事件名.md`（如 `12-18_告白.md`、`02-14_情人节.md`）
- **多日合并文件**：`MM-DD至MM-DD_事件名.md`（如 `11-24至11-28_幸村觉醒一周.md`）
- **跨月/跨周块**：`MM月_事件名.md` 或 `MM月-MM月_事件名.md`（早期日常段落）

### 每个 day 文件结构

文件最前面是 **frontmatter**（YAML-like 块，三短线包围），含：

```yaml
---
日期: ...
事件: ...
当日小节: 按时间列每个 sub-scene
在场: 所有出场人物
不在场（重要⚠️）: ★防错关键★ 标注谁本来"看起来该在"但实际不在
当日好感度变化: 数值变动 + 原因
关键金句: 该日最具引用价值的台词原文
反事实警示: ★防错关键★ AI 容易写错的细节
关联前置/后续: 链接到前后 day 文件
物品: 重要物品出现
---
```

## 五、场景结束后 AI 必须做的事（**不可省略**）

### 1. 更新 `memory/affinity.md`
- 找到该角色行，把数字改成新值
- 在文件末尾"变更日志"区域追加一条记录

### 2. 归档到 `scene-log/`
- 文件命名：`scene-log/YYYY-MM-DD_简短场景名.md`
- 同时把这一段正文**也**追加到 `volumes/` 对应卷的对应日文件末尾（双写）
- 如果当日还没有 day 文件 → 新建 `volumes/volN/MM-DD_xxx.md`，加完整 frontmatter

### 3. 更新索引
- `memory/timeline-index.md` 追加一行
- 若引入新的"专名事件"，`memory/events-index.md` 也追加一行

### 4. 更新角色档案（如有新设定）
- 新角色登场 → 加段
- 新偏好/新创伤层 → 加段
- **不要重写**已有内容·只追加

## 八、最常见的脑补陷阱（违反铁律一）

1. **凭记忆引用伏笔**——总会错。先查 events-index.md
2. **看到 outline header 就写细节**——例如看到 `### 一、抵达幸村家门口` 就脑补陶雨鞠躬细节。**必须读全文**
3. **把"看起来该在"的人写进场**——如奶奶 12.18 实际不在家（茶道教室·晚饭前才回），但 AI 容易把她写进温室对话或玄关迎客。frontmatter 的"不在场（重要⚠️）"是最权威的护栏
4. **写"陶雨爱国口号"**——她的中国根脉是历史纵深 / 集体意识 / 血脉传承，不是政治表态
5. **写幸村"霸总命令式"**——他说"你会答应的吧"，底下是"我已经读到你愿意了"
6. **写斋藤"心机表演被拆穿"**——她从不"被拆穿"，她是清醒的主动表演者
7. **写迹部"傲娇"**——他是张扬正面笑，不傲娇。即使输了昂头走
8. **写真田对陶雨"暧昧"**——他已经退让为深度保护，不要回头
9. **忘了更新 affinity.md**——这是这套系统最容易失败的地方
