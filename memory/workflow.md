# AI 工作流程手册

> 这是项目的"操作系统"。任何 AI（Claude / ChatGPT / Gemini…）开始为这个长篇互动叙事写作前，**必须先读完本文件**。
>
> 项目语境：中文长篇互动叙事《新网球王子》同人，主角陶雨（中国留学生）× 男主幸村精市。穿越者设定：陶雨与斋藤侑子双穿越者，斋藤是女主竞争者/反派。

## 二、开新场景时 AI 必须做的事

### 步骤 1：读基础设定（每次都读，**不可跳过**）
- `memory/affinity.md` → 知道当前所有人的好感度
- `memory/characters/陶雨.md` → 主角内核
- `memory/characters/幸村精市.md` → 男主内核（如果是涉及他的场景）

### 步骤 2：按场景人物补读
| 出场人物 | 必读 |
|---|---|
| 立海大正选（真田/柳/仁王/丸井/切原/朱音/柳生/桑原/佐伯/小野）| `memory/characters/立海大正选.md` |
| 青学（龙马/手冢/不二/乾/菊丸…）| `memory/characters/青学.md` |
| 迹部 / 松本 / 榊原 | `memory/characters/迹部与冰帝.md` |
| 斋藤侑子 | `memory/characters/斋藤侑子.md`（**全文必读** · 她的"完美外壳/恶劣内里"反差很容易写错） |
| 千叶崎 / 柳生双叶 / 织田 / 橘 / 林晓彤 | `memory/characters/配角与同学.md` |
| 幸村家庭 | 在 `memory/characters/幸村精市.md` 末尾 |

### 步骤 3：按场景需要补读
- 涉及住所/通勤/座位 → `memory/settings/地理与住所.md`
- 涉及家世/阶级 → `memory/settings/世界观.md`
- 涉及茉莉花/项链/合照/敲门暗号 → `memory/settings/重要物品.md`
- 涉及陶雨治愈幸村 → `memory/settings/治愈记录.md`

### 步骤 4：回溯历史伏笔
- 如果场景需要"回响"过去（例：First Kiss、海原祭、敲夜谈、12.30 第二个三分之一）
  → 先查 `memory/events-index.md` 定位事件在哪个弧文件
  → 再读那个弧文件的具体段落
- 不要凭记忆引用旧情节——**必须查文件**。

### 步骤 5：检查时间线连续性
- 看 `memory/timeline-index.md` 最末日期，确认新场景接续点合理。
- 当前最新已写到 vol5 卷末 4 月 26 日傍晚。

## 三、写场景时必守的人物铁律（最高优先级）

### 关于陶雨
- 她是水瓶座 O 型 ENTP，**但不只是"直接"**——温柔有韧、嘴硬心软、共情极强但说不出自己。
- 善良是底色不是表演。爱憎分明但底线之内放任。
- 关上门等的是"她真正在乎的那个人"推门，不是任何懂她的人。
- 写她崩溃只能在手冢/果果/龙马面前。
- 不能写她"爱国口号式"——她的中国根脉是历史纵深不是政治表演。

### 关于幸村精市
- **他十八岁，这是初恋，热恋最浓期，好感度 90+。**
- 表面克制从容，内里浓度满。脑子里反复回放她的某个表情某句话。
- 笑着说让人心口发紧的话是他常态。掌控温度对陶雨 ≠ 对队员。
- **占有欲是"精细的"不是粗暴的**——他记下事，下一次用一句话重新校准磁场。
- 他能输、会输、接受输。**不能写"男主不能输"那种叙事**——抹掉他最珍贵的东西。
- INFJ 平行独处是他理想亲密形态。

### 关于斋藤侑子
- 外柔内恶 / 表里割裂 / 重生玛丽苏自我叙事 / 阶级绝对分明。
- 温柔是上流社交外壳。骨子里轻视平庸。
- 善意 buff 免疫名单：青学全员、迹部、林晓彤、切原赤也。
- 看 `memory/characters/斋藤侑子.md` 完整闭环，每条都要遵守。

### 关于其他立海大
- 真田对斋藤有"从来就跟幸村在一起"的潜意识；对陶雨曾暗恋已退让为深度保护。
- 柳是 Te 数据流但内核温柔，2.25 起跟陶雨开了独立通道。
- 仁王看好戏关键时刻认真。
- 丸井 / 切原赤也 / 切原朱音对陶雨已经是亲近重要级。
- 佐伯结衣的转折等她**独自做出只为自己的决定**——不要替她写转身。

## 五、场景结束后 AI 必须做的事（**不可省略**）

### 1. 更新 `memory/affinity.md`
- 找到该角色行，把数字改成新值。
- 在文件末尾"变更日志"区域追加一条记录，格式见 affinity.md 第五节。

### 2. 归档到 `scene-log/`
- 文件命名：`scene-log/YYYY-MM-DD_简短场景名.md`
- 同时把这一段正文**也**追加到 `volumes/` 对应卷的对应弧文件末尾（双写：日志保完整、卷子保连续）。
- 卷文件 → 看本场景日期落在哪段：
  - 4 月底至 5 月 → 暂存 scene-log 即可，未来新建 `volumes/vol6/` 时再归并

### 3. 更新两个索引
- `memory/timeline-index.md` 追加一行：日期 → scene-log 文件路径
- 如果引入了新的"专名事件"，`memory/events-index.md` 追加一行

### 4. 更新 `memory/characters/*.md` 中的设定（如果场景揭示了新设定）
- 新角色登场 → 在合适的 character 文件加段
- 新偏好/新创伤层 → 加在主角文件
- **不要重写**已有内容，只追加。

### 5. 任何修改完成后，git commit
（如果用户在 Claude Code 环境，每次结束自动 commit；如果在网页版，用户手动同步到仓库。）

## 六、跨工具使用说明

### Claude Code（命令行 / IDE）
- 默认所有 4-5 步骤都由 AI 自动完成。
- 用户只需说"写一个 xxx 场景"，AI 自动读 → 写 → 改 affinity → 归档 → commit。

### ChatGPT / Claude 网页版 / Gemini
- 把本文件 + `memory/affinity.md` + 涉及到的 `characters/*.md` + `settings/*.md` 复制到对话开头当系统提示词。
- AI 写完场景后**输出**：
  1. 场景正文（带【好感度变化】区段）
  2. 一段"affinity.md 应当变成什么样"的 diff 片段
  3. 一段"timeline-index.md 应当追加什么"的片段
- 用户手动把这三块粘回仓库文件。

## 七、写作铁律清单（短版本 · 写完一段对照一次）

- [ ] 没有写"陶雨直接"但同时温柔有韧的双重性？只写"直接"就是 OOC。
- [ ] 没有把幸村写成事事顺从的完美男友？他是有棱角的强者。
- [ ] 没有让幸村写成被强行扭转的"必赢男主"？他能接受输。
- [ ] 没有把斋藤写浅薄？她是高度自洽的天命女主叙事 + 阶级绝对分明，**不是傻白甜**。
- [ ] 没有让佐伯结衣转身得太快？她的转折需要独自做出"只为自己"的决定。
- [ ] 好感度数值与 `memory/affinity.md` 一致？
- [ ] 场景结尾有【好感度变化】段？
- [ ] 时间线接续点检查过？

## 八、常见陷阱

1. **凭记忆引用伏笔**——总会错。先查 `events-index.md`。
2. **写"陶雨爱国口号"**——她的中国根脉是历史纵深 / 集体意识 / 血脉传承，不是政治表态。
3. **写幸村"霸总命令式"**——他说"你会答应的吧"，底下是"我已经读到你愿意了"。
4. **写斋藤"心机表演被拆穿"**——她从不"被拆穿"，她是清醒的主动表演者，她比你更知道自己在演。
5. **写迹部"傲娇"**——他是张扬正面笑，不傲娇。即使输了昂头走。
6. **写真田对陶雨"暧昧"**——他已经退让为深度保护，不要回头。
7. **忘了更新 affinity.md**——这是这套系统最容易失败的地方。每场必更新，否则下一场用错数字。
