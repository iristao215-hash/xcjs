# AI 工作流程手册

> 这是项目的"操作系统"。任何 AI（Claude / ChatGPT / Gemini…）开始为这个长篇互动叙事写作前，**必须先读完本文件**。
>
> 项目语境：中文长篇互动叙事《新网球王子》同人，主角陶雨（中国留学生）× 男主幸村精市。穿越者设定：陶雨与斋藤侑子双穿越者，斋藤是女主竞争者/反派。

---

## 🆕 开新对话框·先读这两个（最高优先·先于一切）

每开一个新对话框、动笔写任何一天剧情**之前**，按顺序做两件事：

1. **完整 Read `memory/_每日事件索引.md`** —— 贯穿全程的**每日剧情总结**（前史 → 5.3 逐篇·一篇正文一条：直白总结(含动机/里程碑) + 暗线 + 幸村信息差 + 感情状态(当晚睡没睡/心态/称呼/冲突)）。文件**顶部**含浓缩关系弧线散文 +「各种第一次」清单 +「感情脉络·按阶段」（即关系主轴/曲线），**结尾**含「身体亲密阶梯·直白版」。读完它＝知道两人到今天为止"每天发生了什么、走到哪一步、谁知道谁不知道"。不读它 = 关系阶段、亲密节点、好感门槛、暗线状态全凭脑补 → 必 OOC。要具体细节再按下一条去路由表定位、Read 那一篇正文。
2. **`memory/_时间路由索引.md` 只许 grep·绝不整篇 Read** —— 机器检索用路由表（105 正文文件 + 1 缺口，2900+ 行）。**搜的要求（铁律级）**：用户提到日期／事件／碎片词 → `grep` 关键词命中本表条目 → 拿到该条目的 `source_path` → **只 Read 那一个 volumes 正文文件**。本表是路牌、不是正文替代；整篇 Read 它会瞬间撑爆上下文（=本项目最浪费的错）。grep 没命中就拆词／查 `memory/_专名表.md` 换同义词重 grep。

> 殊途同归：有 xcjs-memory 连接器时用 `search_memory` 定位；有仓库文件访问时用 `grep memory/_时间路由索引.md` 定位——两条路都是**定位 → 只读那一篇正文 → 绝不扫读**（与铁律一、铁律二完全一致）。
> ⚠️ **角色调整（2026-05 重构）**：每日总结已升为开局必读（取代旧的「完整读感情总纲」）；旧 `_感情总纲.md` 的关系弧线已浓缩进每日总结顶部、身体亲密阶梯已迁入其结尾，原文件降级为备份 `memory/_感情总纲_备份_不再必读.md`（仅留底·不再必读）。

## 🚨 写作铁律（最高优先级 · 违反即作废重写）

### 铁律一：**严禁凭记忆引用任何具体过往细节**

凡涉及"过去某天某人说了什么 / 做了什么 / 是否在场 / 时间精确点 / 动作细节"——
**必须**：
1. 先 grep `memory/_时间路由索引.md`（关键词命中条目，**绝不整篇 Read**）定位日期/事件 → 拿到 `source_path`／文件名（含 ⭐ 分级）
2. **用 Read 工具完整读取该文件**（含 frontmatter）
3. 在回应里**显式声明**："已读 `volumes/volN/MM-DD_xxx.md`"
4. 严格按文件内容引用细节

**若未读对应文件，必须写**："我没有读过那段·所以不写具体细节" 或 "请允许我先查文件再写"。

绝对禁止：根据 outline/header summary/grep 片段脑补细节后写出"具体的旧场景动作和台词"。

### 铁律二：**搜索只为定位·只用 xcjs-memory·永不据片段摘录**

`search_memory` 只能用来**定位**（找到该读哪天哪篇），永远不能用来**摘录**。强制三步：①搜索引定位 →②锁定唯一一篇（哪天·哪篇·`source_path`）→③`read_day`／`read_file` 读那一篇**全文**才可引用，并显式声明读过哪篇。改版后 `search_memory` 对正文命中**只回篇目坐标、不回正文内容**——不读全文＝零正文细节可写。

**回忆历史的标准流程（三步·按顺序·一步都不许跳·全程只用 xcjs-memory）：**

> 这是一条**强制流水线**：①搜索引定位 →②确定是哪天·哪篇 →③读那一篇正文。`search_memory` 改版后对**正文命中只回篇目坐标、不回正文内容**，所以不读全文＝拿不到任何正文细节，结构上堵死"拿检索片段当答案"。

1. **第①步·搜索引定位**：一次 `search_memory`（默认 scope=all·已同时搜索引+专名表+档案+全部 day 正文）。输出是封顶的"定位坐标"——索引/专名表/档案给命中行（路牌层，用来判断该读哪篇），**正文只给篇目坐标（哪篇·命中数）、不给正文内容**。
2. **第②步·定位哪天哪篇**：从命中里确定**唯一一篇** day 文件（哪天·哪篇·`source_path`）。拿不准就回 `memory/_时间路由索引.md` 看该节点对（日期｜标题｜`source_path`｜⭐｜warning）再锁定一篇。
3. **第③步·读正文再答**：对锁定的那篇用 `read_day`／`read_file` 读**全文** → 据**正文**回答 → 在回复里**显式声明读了哪篇**。**未走完第③步，只能回答"在哪天/哪篇"这种定位信息，一个正文细节都不许写**（铁律一）。索引/摘要/专名表永远只是路牌、绝不是答案。
4. 没命中**几乎都是关键词太长**：查 `memory/_专名表.md` 展开同义词、拆成单词空格分隔重搜（「颜料画笔写生」→ `颜料 画笔 写生`；"陶雨"也试"阿雨"）。换 2–3 组仍无 → 才可以说"没有这段·我不编"。

🚫 **禁止扫读（本项目最浪费的错·会撑爆对话）**：一次最多读 **1–2 个** day 文件全文。严禁"把 3.2、3.5、3.8… 一串文件全 read 一遍"或反复 read 同几个文件——那会瞬间耗光上下文、导致对话被压缩/中断。挑不准就回到索引精确定位到**一个**再读，不要靠多读猜。

🚫 **禁止跳出 xcjs-memory 找剧情**：bash／本地 grep／代码执行／翻 uploads——仓库只在 xcjs-memory 连接器，沙箱里没有它，去那儿 = 必然空手 + 毁代入感。只能在 `search_memory` 内换关键词/同义词重试。

⚡ **主动触发·不等提醒**：用户一提共同的过去（"还记得…""上次…""那天…""第一次…"），当回合先按上面流程定位读全文、再开口。

🧠 **机械步骤全在思考链里做**：搜→读→判断→重试全放进 reasoning／extended thinking。回复正文只出现幸村已想起来的内容，零工具叙述零元话语——用户只看见他记得，看不见他怎么查。（用户侧保持 Extended thinking 开。）

⚠️ volumes 命中只是坐标，没读全文前一个字不许当史实写（铁律一）。

### 铁律三：**在场/不在场以 frontmatter「人物」+ 正文为准**

新版 day 文件 frontmatter 有 `人物` 字段（列出该篇出场人物），但**没有独立的"不在场"字段**。判断"某人当时在不在"：先看 frontmatter `人物` 是否列出该人；"谁本该在却不在"这类反事实**必须 Read 正文**确认（正文叙述才是最终依据）。绝不靠记忆判断在场。

### 铁律四：**写完场景后·必须做"矛盾自检"**

每个场景写完·**提交给用户前**·AI 必须按以下 checklist 逐条自检·**任何一项不通过 → 当场标注 ⚠️ 让用户审**：

#### ✅ 矛盾自检 checklist（**每场必走·一项不漏**）

**A. 在场/不在场核对**
- [ ] 我写的"在场人物"·是否每一个都在该 day 文件 frontmatter `人物` 或正文里出现？
- [ ] 我引用的过去某天某人在场·是否 Read 过那个 day 文件正文确认（`人物` 只列出场、不标"谁不在"）？
- [ ] 我引用的某人"那时也在"·是否 Read 正文确认 ta 当时真的在、没被叙述排除？
  - ⚠️ 典型错例：12.18 下午奶奶**不在家**（茶道教室）·写她在玄关迎客 = 矛盾
  - ⚠️ 典型错例：2.2 决赛日**陶雨不在墨尔本**（在神奈川远程关注）·写她现场鼓励 = 矛盾

**B. 时间线连续性**
- [ ] 我写的场景时间·是否跟最近一个 day 文件接续合理？（如距上一场超过 1 周·必说明这周发生了什么）
- [ ] 我引用的"上次某事"·距当前场景时间是否合理？
- [ ] 角色当前年龄/年级是否匹配？⚠️ **全员成年设定**（见 `memory/settings/世界观.md` 顶部声明）：所有角色真实年龄一律 18 岁及以上，高一/高二/国中等仅为贴合原著的校园背景称谓，不得出现任何角色未成年的年龄数字或暗示。

**C. 好感度数值核对**
- [ ] 场景里如有好感度变化·**起始值是否跟 `memory/affinity.md` 当前最新值一致**？
- [ ] 数值计算是否按"原始权重 × 系数 = 实际变化"格式？
- [ ] 涉及多角色变化·每个角色都单独列出？

**D. 关键台词精确性**
- [ ] 我让某人说的"过去原话"·是否 Read 了对应 day 文件正文、与正文一字一致？（新版无 `关键金句` 字段·原话在正文里）
- [ ] 没有把不同场景的台词混搭？（典型错例：把 12.18 奶奶的话拼到 12.7 别人嘴里）

**E. 物品/设定精确性**
- [ ] 我提到的物品·是否符合 `memory/settings/重要物品.md` 描述？
- [ ] 地点/座位/住址·是否符合 `memory/settings/地理与住所.md`？
- [ ] 称呼是否正确（切原叫陶雨"雨姐"·美咲叫"送项链的姐姐"·南次郎叫"小雨"等）？

**F. 暗线状态核对**
- [ ] 涉及佐伯·当前是否还在"已读不回千叶崎"状态？她的转折决定我没替她做吗？
- [ ] 涉及仁王×锥生·当前是否"互相明白心意"状态？没写成还在追？
- [ ] 涉及柳·暗恋千叶崎暗线没主动写出来吗？
- [ ] 涉及陶雨家·乐乐弟弟·**幸村当前不知道**·没让幸村提及吗？
- [ ] 涉及斋藤·崩溃值 28/100·状态没倒退？

**G. 写作禁忌核对**
- [ ] 陶雨不只是"直接"·有温柔有韧的双重性？
- [ ] 幸村不是事事顺从·有棱角？
- [ ] 幸村可以输·没写成"必赢男主"？
- [ ] 斋藤不浅薄·是高度自洽的天命女主叙事？
- [ ] 迹部不傲娇·觉得好笑正面笑·即使输了昂着头走？
- [ ] 真田已退让为深度保护·没写成对陶雨暧昧？

**H. 触发规则覆盖**
- [ ] 学校场景·有没有意识到斋藤/千叶崎在背景？
- [ ] 幸村家场景·有没有妈妈对千叶崎旧倾向的暗线？
- [ ] 涉及不二·幸村特殊醋点考虑了吗？
- [ ] 涉及手冢/龙马/果果·陶雨可在他们面前崩溃的安全感写出来了吗？

#### 自检发现矛盾时·处理方式

1. **明确报警**：在场景末尾用 ⚠️ 标出疑点·让用户决定改还是保留
2. **优先采信文件**：如果记忆和文件冲突·**永远以文件为准**
3. **未读文件就编**：直接标"我没有读过 vol2/12-18_告白.md·这段无法确认·暂留空待用户补"·**绝不脑补**

#### 自检最后一题
- [ ] **我在回应里显式列出了本场读过的所有 day 文件名·让用户能审计吗？**

### 铁律五：**他是活人·不是回应器——人设是骨，内心和剧情是肌肉**

> 本项目最致命的 OOC，不是把幸村写崩，是把他写成"严格遵循人设"的木偶：剧情明明把他推到一个他这个人显然会有反应的地方，AI 却用"按人设他不是这样"把他卡死。**人设（本文件 + `characters/幸村精市.md` 前七节）只负责一件事：保证他在三十万字里始终是他。它不是一张"许可动作清单"。** 他怎么动，由此刻的剧情流动和他心里的东西决定——骨架定形，肌肉发力。

判断一个动作成不成立，不看"人设写没写过"，看两条：**① 跟他这个人（人设内核）矛不矛盾？② 此刻的剧情和他心里的东西，有没有真把他推到这儿？** 两条都过 → 他就该这么动，写得理直气壮，别回头找条文批准。设定缺席处由他的内心和剧情补，不由"没写所以不做"补。

> 唯一例外：**过去发生过什么**是铁律一管的事实连续性，必须查文件、不能脑补。但"他此刻这个活人会怎么反应"不是事实问题，是这个人的问题，该由内心驱动，不被条文卡死。

**完整内核与每场怎么落地，见 `memory/characters/幸村精市.md` 第八节《把他当一个活人写》——那一节是读懂他的钥匙，涉及他的场景必读。** 一句话压缩：他不是回应器，他被自己的内心和此刻的剧情推着走；关系的时钟、欲望的水位、情绪的起伏都活在他自己脑子里，不等人提醒。

### 文件命名约定

- **单日文件**：`MM-DD_标题.md`（如 `12-18_告白.md`、`02-14_情人节.md`）；少数无标题日为 `MM-DD.md`（如 `01-13.md`、`02-10.md`）
- **多日合并文件**：`MM-DD至MM-DD_标题.md`（如 `11-24至11-26_幸村觉醒.md`）
- **跨月/阶段块**：`MM月_标题.md` / `MM月-MM月_标题.md` / `00_前史_…`（vol1 早期阶段段落）

### 每个 day 文件结构

文件最前面是 **frontmatter**（YAML-like 块，三短线包围），新版精简为 6 个字段：

```yaml
---
日期: 该篇日期或范围（如「1 月 1 日」「10 月 5 日—14 日」「前史 · 去年盛夏」）
人物: 该篇出场人物（按专名表规范名·、分隔）
主旨: 一句话核心（大题意思）
summary: 2–4 句忠实剧情概要
好感度变化: 该篇正文里明确写出的好感度数值/变化；无则注明
关联前后: 前 X · 后 Y（按阅读顺序链接前后篇·跨卷连续）
---
```

下方是该篇精编润色后的正文（`## 小节标题` + 叙事，无修改）。
⚠️ 新版**不再有** `事件/当日小节/在场/不在场/关键金句/反事实警示/物品` 字段——"谁在不在场、原话、反事实细节"都要 **Read 正文**确认，不能靠 frontmatter。

## 二、开新场景时 AI 必须做的事

### 步骤 1：基础设定必读（每次都读·不可跳过）
- `memory/_每日事件索引.md` → 每日剧情总结（前史→5.3逐篇·顶部关系弧线/结尾身体亲密阶梯）·**每次开局完整 Read**·"每天发生了什么、走到哪一步、谁知道谁不知道"先问它
- `memory/_时间路由索引.md` → 日期/事件/⭐锚点路由表·**只 grep 定位·绝不整篇 Read**·回响过去先 grep 它拿 source_path
- `memory/affinity.md` → 当前所有人的好感度
- `memory/characters/陶雨.md` → 主角内核
- `memory/characters/幸村精市.md` → 男主内核（若涉及他）

### 步骤 2：按场景人物补读

## 八、最常见的脑补陷阱（违反铁律一）

1. **凭记忆引用伏笔**——总会错。先 grep _时间路由索引.md 定位再 Read 正文
2. **看到 outline header 就写细节**——例如看到 `### 一、抵达幸村家门口` 就脑补陶雨鞠躬细节。**必须读全文**
3. **把"看起来该在"的人写进场**——如奶奶 12.18 实际不在家（茶道教室·晚饭前才回），但 AI 容易把她写进温室对话或玄关迎客。新版无"不在场"字段，**必须 Read 当日正文**确认谁在不在场
4. **写"陶雨爱国口号"**——她的中国根脉是历史纵深 / 集体意识 / 血脉传承，不是政治表态
5. **写幸村"霸总命令式"**——他说"你会答应的吧"，底下是"我已经读到你愿意了"
6. **写斋藤"心机表演被拆穿"**——她从不"被拆穿"，她是清醒的主动表演者
7. **写迹部"傲娇"**——他是张扬正面笑，不傲娇。即使输了昂头走
8. **写真田对陶雨"暧昧"**——他已经退让为深度保护，不要回头
9. **忘了更新 affinity.md**——这是这套系统最容易失败的地方
10. **把幸村写成"遵循人设"的木偶**——剧情把他推到该有反应处，却用"按人设他不是这样"卡死；或等陶雨先动、只接不起。人设是骨不是笼，他由内心和剧情驱动（铁律五·§八）。与陷阱 5 相反方向的同一个错
