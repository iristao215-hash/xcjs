1. 全局比对发现矛盾

## 工作流·两阶段

### 阶段一: 一致性审计 (先做·不动剧情)

1. 读完所有源材料 (6 个 txt + memory/ 现有结构)
2. 产出 `_一致性审计报告.md` 放在 repo 根目录·内容:
   - § 1 已拍板 retcon 的影响清单 (住址 / 陶雨动机 / 初三初见 → 涉及哪些 day 文件)
   - § 2 陶雨动机暗线·完整 reveal 弧线整理 (按时间顺序列所有相关时刻)
   - § 3 疑似矛盾清单 (每条带原文位置 + 推荐方案 + 备选)
   - § 4 压缩策略草案 (哪些类型的内容标记为可压·示例)
3. commit + push·让作者在 GitHub 上看·或者用 MCP `read_file` 读

**等作者拍板后再进阶段二·不要先斩后奏。**

### 阶段二: 压缩 + 修复 (按拍板方案推进)

按 day 顺序从 vol1 推到 vol5。每个 day 文件:

1. 读原 day 文件 (`volumes/volX/YY-YY_xxx.md`)
2. 应用所有已拍板 retcon (住址 / 动机暗线 / 其他)
3. 压缩到 ~45%·遵循:
   - ✅ **全保**: 所有事件·关键动作·结构性进展
   - ✅ **全保**: 金句 (告白·重要承诺·人物标志性台词原文一字不改)
   - ✅ **全保**: 好感度变化点·情感转折
   - ✂️ **删**: 重复抒情 (同一心情连写三遍那种)
   - ✂️ **删**: 冗余环境描写 (天气·景色·路上没事说半页那种)
   - ✂️ **删**: 与主线无关的闲笔 / 过场水描写
4. 输出到 `volumes_tight/volX/YY-YY_xxx.md` (镜像原结构)
5. 每完成一卷·commit + push·让作者抽查
6. 顺手更新 `memory/characters/`·`memory/_每日事件索引.md`·`memory/affinity.md` 等设定库

**不要一次性输出几万字让作者审·分批 commit·让作者用 git diff 或 GitHub 网页对比看。**
